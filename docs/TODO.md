2. 添加在线获取歌曲标签的功能：http://music.163.com/api/search/get/?s=<关键词>&limit=<数量>&type=1&offset=0
3. 设置项扩展：
   - 添加规则（点击添加文件/扫描目录时是叠加还是覆盖，单文件和目录分别设置）
   - 艺术家分隔符（多个艺术家时重命名用什么符号分割，默认 _）
   UI 交互：
   - 左侧边栏添加主页按钮、添加设置按钮
   - 主页与设置页面平滑进入/退出切换(符合当前动画)
4. 音频规格转换降采样：
   - 核心逻辑：
     - 自动检测音频采样率与编码格式；位深仅在"无损输入→无损输出"链路中参与判断，
       对有损输入格式（MP3/AAC/OGG/Opus/WMA）不做位深判定
     - 支持手动选择目标规格，仅对高于目标规格的音频进行降采样/降位深处理，
       低于或等于目标规格时自动跳过，不支持升频/升位深
   - 输出为 FLAC / WAV 时，支持目标规格（仅对无损输入有效）：
     - 48kHz / 24bit
     - 44.1kHz / 24bit
     - 44.1kHz / 16bit
   - 输出为 MP3 时：
     - 固定目标采样率为 44.1kHz（若输入采样率高于 44.1kHz，如 48kHz，则自动重采样）
     - 位深不作为 MP3 的输出规格（MP3 为有损编码，不以 PCM 位深存储）
     - 仅可选择目标比特率
     - 若输入已为 MP3 且采样率为 44.1kHz 且比特率相同，则自动跳过
   - 输入输出格式对应规则：
     - 输出为 FLAC / WAV / ALAC（.m4a）（无损格式）：
       仅支持无损格式输入（FLAC / WAV / ALAC），其他有损格式（MP3 / AAC / OGG /
       Opus / WMA）直接跳过，不做无损转码
     - 输出为 MP3（有损格式）：
       兼容所有输入格式转码，若输入采样率高于 44.1kHz 则自动降采样至 44.1kHz
   - 跳过条件汇总：
     - 有损输入 → 无损输出：直接跳过
     - 任意输入采样率 ≤ 目标采样率 且 位深 ≤ 目标位深（无损链路）：跳过
     - MP3 输入采样率已为 44.1kHz 且比特率与目标一致：跳过
   - 输入格式支持：
     - 无损：FLAC、WAV、ALAC（.m4a）
     - 有损：MP3、M4A / AAC、OGG / Opus、WMA
   - 输出格式支持：
     - FLAC
     - WAV
     - ALAC（.m4a）
     - MP3
   - 技术实现：
     - 桌面端（Windows / macOS / Linux）使用 ffmpeg_cli + 随应用打包 ffmpeg / ffprobe 二进制
     - 使用 ffprobe 自动检测 codec_name / sample_rate / bits_per_raw_sample
     - ffprobe 位深检测回退逻辑：优先 bits_per_raw_sample → sample_fmt 推断（s16/s24/s32）→ 默认 16
     - 重采样使用 soxr 引擎（-resampler soxr）保证音质
     - 降位深时可选启用 triangular 抖动（实际听感差异极小，可配置）
     - ALAC 编码优化：-map_metadata 0 保留元数据（封面/歌词），显式标注 sample_rate 解决部分播放器识别异常
     - 技术实现不依赖 ffmpeg_kit_flutter
   - 桌面端适配：
     - 支持单文件与批量文件夹处理
     - 异步执行任务，不阻塞 UI
     - 实时显示处理进度（解析 ffmpeg stderr 时间戳推算进度）
     - 自动跳过已符合目标规格的文件，并在任务列表中标注跳过原因
   - 处理策略：
     - 一个新的批量处理时限制并发数常量（建议同时执行 1-2 个任务），降低系统资源占用（支持手动调整上限为5）
     - 处理完成后支持：保留原文件 / 替换原文件 / 输出到指定目录
   - 代码复用：
     - 优先使用可复用的 UI 组件与模块、常量，避免重复实现
     - 按层拆分：ProbeService（探测）/ RuleEngine（决策）/
       CommandBuilder（构建命令）/ TaskQueue（队列与进度）
