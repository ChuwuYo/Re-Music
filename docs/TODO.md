1. 添加封面修改与保存/在线获取
2. 添加在线获取歌曲标签的功能：http://music.163.com/api/search/get/?s=<关键词>&limit=<数量>&type=1&offset=0
3. 设置项扩展：
   - 添加规则（点击添加文件/扫描目录时是叠加还是覆盖，单文件和目录分别设置）
   - 艺术家分隔符（多个艺术家时重命名用什么符号分割，默认 _）
   UI 交互：
   - 功能栏左侧添加主页按钮
   - 功能栏右侧添加设置按钮
   - 主页与设置页面平滑进入/退出切换
4. 音频规格转换降采样：
   - 核心逻辑：自动检测音频采样率与位深，支持手动选择目标规格，仅对高于目标规格的音频进行降采样处理，低于或等于目标规格时不处理，不支持无损升频
   - 输出为 FLAC / WAV 时，支持目标规格：
     - 48kHz / 24bit
     - 44.1kHz / 24bit
     - 44.1kHz / 16bit
   - 输出为 MP3 时，自动固定为 44.1kHz / 16bit 规格，不支持高规格，仅可选择比特率
   - 输入输出格式对应规则：
     - 输出为 FLAC/WAV/ALAC（.m4a）（无损格式）：仅支持无损格式的输入格式转换，其他有损输入格式直接跳过
     - 输出为 MP3（有损格式）：兼容所有输入格式直接转码，自动适配 44.1kHz / 16bit 规格
   - 输入格式支持：
     - MP3
     - FLAC
     - ALAC（.m4a）
     - M4A / AAC
     - OGG / Opus
     - WMA
     - WAV
     - DSF
     - DFF
   - 输出格式支持：
     - FLAC
     - WAV
     - ALAC（.m4a）
     - MP3
   - 技术实现：集成 ffmpeg_kit_flutter 实现重采样，保证音质
   - 桌面端适配：
     - 支持单文件与批量文件夹处理
     - 异步执行任务，不阻塞 UI
     - 实时显示处理进度
     - 自动跳过已符合目标规格的文件
   - 处理策略：
     - 优先使用 FLAC 输出以保证速度与音质
     - 批量处理时分批执行，降低系统资源占用
     - 处理完成后支持保留原文件、替换原文件或输出到指定目录
   - 功能代码复用
   - 优先使用可复用的UI组件与模块、常量，避免重复实现
